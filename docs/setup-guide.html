Hosting & Maintaining the Resident
Reference Dashboard

A step-by-step guide — no coding required

Prepared for Dr. Rachel Brancamp, Associate Director, Transfusion Medicine.
Goal: put the dashboard on your own GitHub, publish it as a live link for residents, and update it going forward by simply describing changes in plain English to Claude Code. Everything below is done in a web browser.

The dashboard is a self-contained website (a set of files). GitHub stores those files for free and publishes them as a live web page (a feature called GitHub Pages). You will do three things, once: (A) copy the files to your GitHub, (B) publish them, and (C) connect Claude Code so you can maintain it anytime. Plan about 30 minutes for the one-time setup.

What you'll have at the end

  • Your own free GitHub account with your own copy of the dashboard.
  • A live link — https://YOUR-NAME.github.io/… — to share with residents.
  • Claude Code connected to it, so you can update content by typing a request in plain English.
  • Full independence — it runs under your account and does not depend on anyone else.

Before you begin — a quick checklist

  • A computer with a web browser (Mac or Windows). Nothing to install.
  • A GitHub account — free (created in Part A).
  • A Claude Pro subscription — about $20/month, which includes Claude Code (set up in Part B). GitHub and hosting are free.
  • The source dashboard link: https://github.com/omarzbaba/transfusion-medicine-resident-reference

Part A · Put the dashboard on your GitHub one-time

A1. Create a free GitHub account

  1. Go to github.com and click Sign up.
  2. Enter your email, a password, and a username. (The username becomes part of your web address — pick something clean, e.g. brancamp-tm or hfpath.)
  3. Verify your email. Done. (Already have an account? Skip to A2.)

A2. Make your own copy ("Fork")

  1. Signed in to GitHub, open the source project:
    https://github.com/omarzbaba/transfusion-medicine-resident-reference
  2. Click the Fork button (top-right), then Create fork.
  3. GitHub copies everything into your account. This copy is yours and independent.

Even simpler alternative

Omar can add you as a collaborator or transfer the repository to you directly — then it is already in your account and you can skip the fork. Either way works.

A3. Turn on the free live website (GitHub Pages)

  1. In your copy of the repository, click the Settings tab.
  2. In the left menu, click Pages.
  3. Under Source, choose Deploy from a branch; set branch to main and folder to / (root); click Save.
  4. Wait ~1 minute and refresh. Your live link appears at the top: https://YOUR-USERNAME.github.io/transfusion-medicine-resident-reference/

A4. Share it

Send residents that link. It works on any browser or phone, with no login. That's the hosting done — now let's make it easy to maintain.

Part B · Connect Claude Code one-time · ~10 min

Claude Code lets you change the dashboard by describing what you want — it edits the files and saves them back to GitHub for you. The easiest version runs in your browser (nothing to install).

B1. Get a Claude subscription

  1. Go to claude.com/pricing and choose Claude Pro (~$20/month — plenty for maintaining this dashboard).
  2. Sign up with your email and complete payment.

B2. Open Claude Code

  1. Go to claude.ai/code and sign in with your Claude account.
  2. Prefer an installed app? You can instead download the Claude Code desktop app from the same page — the steps below are identical once you're signed in.

B3. Connect your GitHub

  1. On claude.ai/code you'll see "Connect GitHub." Click it.
  2. Click Install Claude GitHub App. You're taken to GitHub.
  3. Choose which repositories Claude may access — select your transfusion-medicine-resident-reference repo (or "all repositories"), then click Install to authorize.

What you just allowed

You gave Claude permission to read your repository and to write changes (as new branches / pull requests) — nothing more. It cannot touch your computer, and it cannot delete your repository. You can revoke access anytime in GitHub → Settings → Applications.

B4. Create your workspace

  1. When prompted to "Create your environment," give it a name (e.g., TM Dashboard).
  2. Leave the other options at their defaults and click Create environment.

That's the whole setup. From now on you just open claude.ai/code and go.

Part C · Update the dashboard, anytime

This is what you'll actually do day-to-day. No coding — you describe the change in plain English.

  1. Go to claude.ai/code and select your TM Dashboard repository.
  2. In the message box, type what you want changed (see examples below) and press Enter.
  3. Claude reads the files, makes the change, and shows you a before/after preview. Approve with ✓ Accept (or click Explain first to see what it will do).
  4. Click Create PR (pull request) to save the change to GitHub.
  5. On GitHub, open the pull request, review the highlighted changes, and click Merge.
  6. Your live site updates automatically in 1–2 minutes. Refresh to see it.

Example requests — tailored to this dashboard

You: In the Platelet requests tool, change the prophylactic threshold guidance from "under 10" to "under 20."
You: Update the MSBOS: change the crossmatch units for "Craniotomy" to 4, and add a new procedure "Robotic prostatectomy — Type & Screen."
You: Add three new quiz questions about telling TACO apart from TRALI, with explanations.
You: Update the ASFA entry for "Thrombotic thrombocytopenic purpura" to reflect the latest edition.
You: Change the welcome text on the Start Here page, and add our blood bank pager number to the footer.
You: Show me how the platelet tool decides "appropriate" vs "not indicated" before I change anything.

Where the content lives (in case you're curious)

You don't need to open these yourself — just tell Claude what you want and it will find the right place. But it helps to know the refdata.js and abo.data.js files are the clinical knowledge.

⚠ Clinical & safety notes

  • Review clinical changes with the blood bank before residents rely on them — the MSBOS is your institutional policy (HFHS PCR-PALM-TRM-5.050); ASFA follows the 9th ed. (2023).
  • Every change waits for your approval and goes to a separate branch first — nothing changes the live site until you merge. You can always revert.
  • Do not paste passwords or any patient information into Claude.
  • The tool is decision support, not a protocol — the attending / blood bank physician is the final decision-maker (this is stated in the dashboard footer).

Quick reference — your typical update

  1. Open claude.ai/code → pick your dashboard repo.
  2. Type the change you want, in plain English → Enter.
  3. Review the preview → ✓ Accept → Create PR.
  4. On GitHub, review and Merge.
  5. Live site updates in 1–2 minutes. Done.
